# 选片流程设计

本文定义 Keeper 的核心选片流程。所有引擎/业务代码以此为准。

## 名词

- **瞬间组（group）**：一次拍摄里相似的连拍被聚到一起，代表「同一个瞬间」的若干张。
- **漏斗（funnel）**：一层「评分 → 筛选」。Keeper 有两层级联漏斗（本地、在线 LLM），规则相同。
- **N（基础保底数）**：由分组照片数算出，也是 **LLM 层（层②）输出**的张数下限。
- **M（本地保底数）**：**本地层（层①）输出**的张数下限，`M = ceil(1.5 × N)`——比 N 放宽 50%，多送些进 LLM 给它翻案空间。
- **PK / 擂台**：用户在桌面端做 A/B 两两对决、最终选片的环节。

## 核心原则

1. **机器只负责「别让用户没得挑」，绝不替用户「砍掉够好的」。** 最终留谁、是否整组舍弃，是用户在擂台上的权利。
2. **保底数是「进入下一层的下限」，不是「最终归档的下限」。** 用户在擂台里可以把进来的图全部舍弃。

## 整体：两层级联漏斗

分组之后，照片依次穿过两层规则相同的漏斗，最后进入用户 PK：

```
组内全部照片
   │
   ├─【0 分组】DINOv2 语义 + 时间 + 人脸聚类 → 一个个「瞬间组」
   │
   ▼  ┌──────────────── 层① 本地模型漏斗 ────────────────┐   本地 · 全量 · 不上传
   │  │ 本地模型对组内每张打 0–100 分（技术质量为主：      │
   │  │ 锐度 / 曝光 / 人脸 / IQA 美学）。≥60 全进、补足 M。 │
   │  └────────────────────────────────────────────────────┘
   │       │ 输出 ≥ M 张（M = 1.5N）
   ▼  ┌──────────────── 层② 在线 LLM 漏斗 ────────────────┐   云端 · 只传低清预览 · 即焚
   │  │ LLM 对上一层进来的每张打 0–100 分（审美 / 表情 /    │   唯一花钱、唯一联网
   │  │ 构图 / 语义）。≥60 全进、补足 N。                   │
   │  └────────────────────────────────────────────────────┘
   │       │ 输出 ≥ N 张
   └─【用户 PK】A/B 擂台两两对决，用户全权终选（可整组清空）     本地 · 写回结果
```

## 漏斗规则（两层通用，最关键）

每层漏斗对它的输入照片打 0–100 分，按以下规则决定哪些通过到下一层。设这一层输入 K 张、**该层保底数**（层① 为 M，层② 为 N）：

1. **质量主规则**：分 **≥ 60** 的，**全部**通过 —— **不设上限**。
2. **数量兜底**：若 ≥60 的不足保底数，用 <60 分的按分从高到低补。
3. **巧妇难为无米之炊**：通过数不超过输入数 K。输入本就不足保底数（`K <= 保底数`）时，K 张全部通过。

一个公式概括：

```
通过数 = min(K, max(达标数, 保底数))
```

- `K <= 保底数`：全部通过。
- `K > 保底数`、达标数 ≥ 保底数：达标的全过。
- `K > 保底数`、达标数 < 保底数：达标的 + 次高分补到保底数。

一句话：**60 分是「资格线」（达标就过、多少张都过），保底数是「饥荒线」（达标太少时按分补够），输入不够时就全放行。**

> 这条规则在**本地层**和 **LLM 层**完全一样，代码里抽象成一个可复用的 `apply_funnel(scores, n)`；只是两层传入的保底数不同（层① `M`、层② `N`，见「N 怎么定」）。

## 两层的差异（规则同，语义不同）

| | 层① 本地模型 | 层② 在线 LLM |
| :-- | :-- | :-- |
| 输入 | 组内全部照片 | 层①的输出 |
| 评什么 | 技术质量（清晰/曝光/人脸/IQA 美学） | 审美 / 表情 / 构图 / 语义 |
| 在哪算 | 本地，全量，不上传 | 云端，只传低清预览，用完即焚 |
| 成本 | 免费 | 唯一花钱、唯一联网环节 |
| 保底数 | `M = ceil(1.5N)`（放宽） | `N`（收紧） |

**「60 分线」不是同一回事**：本地的 60 是「技术上够格」，LLM 的 60 是「审美上够好」。两层阈值**各自独立校准**（默认都写 60，调参时分开调）。

## N 怎么定

```
N（基础保底数，也是层② LLM 保底数）= max(ceil(组照片总数 × 百分比), 固定值)   # 百分比与固定值「取大」
M（层① 本地保底数）                = ceil(1.5 × N)
```

- **取大**而非 clamp：小组靠固定值兜底（至少给够），大组随百分比放大，**不设上限**。
- 百分比与固定值是可调旋钮（示例：20% 与 3 → `N = max(ceil(total×0.2), 3)`）。
- 层① 用 M（比 N 多 50%），让 LLM 有挑选余地，又不至于全量上云推高成本。
